pygeoapi¶

Web Service¶

pygeoapi est une implémentation serveur en Python de la suite de standards OGC API. Le projet a vu le jour dans le cadre des efforts de la prochaine génération de l’ OGC API en 2018 et permet aux organisations de déployer un point de terminaison d’API OGC RESTful à l’aide d’OpenAPI, GeoJSON et HTML. pygeoapi est open source et publié sous une licence MIT.

Fonctionnalités principales¶

  • Serveur moderne OGC API prêt à l’emploi

  • mise en œuvre conforme et de référence de l’OGC certifiée pour l’OGC API - Features

  • en outre implémente OGC API - Coverages, OGC API - Tiles, OGC API - Processes and SpatioTemporal Asset Library

  • plugins de fournisseur de données prêt à l’emploi pour rasterio, GDAL/OGR, Elasticsearch, PostgreSQL/PostGIS

  • documentation simple d’emploi OpenAPI / Swagger pour les développeurs

  • prend en charge la sortie JSON, GeoJSON, HTML et CSV

  • prend en charge le filtrage des données par requêtes spatiales, temporelles ou attributaires

  • facile à installer : installez une implémentation complète via pip ou git

  • configuration YAML simple

  • facile à déployer : via UbuntuGIS ou l’image officielle de Docker

  • flexible : construit sur un cadre plugin robuste pour créer des connexions, des formats et des processus de données personnalisés

  • prend en charge n’importe quel framework Web Python (inclus flask [par défaut], Starlette)

Standards implémentés¶

Les standards sont au cœur du pygeoapi. Vous trouverez ci-dessous la matrice de supports des normes du projet.

  • Mise en œuvre : mise en œuvre d’une norme (bien)

  • Conforme : conforme aux exigences de conformité de l’OGC (très bien)

  • Mise en œuvre de référence: fournit une référence pour la norme (excellent !)
