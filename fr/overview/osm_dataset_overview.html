OpenStreetMap¶

Données cartographiques¶

OpenStreetMap est une carte du monde alimenté par une « foule de gens » qui a grandit pour devenir une des sources disponible les plus détaillées pour les données à l’échelle locale. Les données source de la carte sont créées et maintenues par des milliers de volontaires autour du monde, en utilisant des processus similaire à ceux servant à la maintenance de l’encyclopédie Wikipédia.

L’aspect le plus visible aspect de OSM est l’interface en ligne avec une carte tuilée de https://www.openstreetmap.org, mais les cartes peuvent aussi être vues, importées, ou éditées dans de nombreuses applications comme QGIS , OpenLayers, ArcGIS et des applications dédiées utilisant OSM.

Le coeur du projet est la donnée sous-jacente qui est ouverte pour éditer, voir ou bien créer des rendus cartographiques personnalisés. Fondamentalement OSM se concentre sur la donnée, les cartes enrichies découlent simplement de ce fait.

La distribution OSGeo-Live inclut un extrait des données de l’échelle d’une ville qui est utilisée pour les exemples par certaines des applications.

Caractéristiques principales¶

  • Objets vecteur sont taggés avec des noms et d’autres attributs.

  • Extraction local d’un sous-ensemble de la donnée.

  • Données stockées sous forme de nœuds contenant des coordonnées en latitude-longitude WGS84, des chaînes de nœuds, et des fonctionnalités d’ordre supérieures permettant de gérer une couverture ponctuelle, linéaire et surfacique.

  • Taggage des attributs riche, souvent bien plus détaillé que beaucoup d’autres sources.

  • Dépôt global de connaissance locale.

Jeux de données inclus dans OSGeo Live¶

  • feature_city.osm.bz2: Un large extrait de la surface de l’agglomération dans un fichier text XML compressé dans un BZip2

  • feature_city_CBD.osm.bz2: Un sous-ensemble couvrant juste le centre-ville.

  • feature_city_poi.db: Base de données Sqlite de nœuds de « positions d’intérêt » extraite de l’extrait OSM cité ci-dessus. Contient des emplacements de pubs, stations-service, restaurants, supermarchés, etc.

  • Les fichiers référençant le nom de la ville en en exergue contiennent les mêmes données que les fichiers ci-dessus. Les guides de démarrage rapide et les aperçus utilisent les noms de fichiers génériques pour que la ville exemple puisse changer à chaque nouvelle version.

Les données de la ville avec les objets géographiques OSM ont été importées dans une base de données PostGIS appelée « osm_local » en utilisant l’outil osm2pgsql. Le système de référence spatiale utilisé pour cette base de données est la latitude-longitude avec le datum WGS84 (code EPSG 4326) et peut être reprojeté dans d’autres Systèmes de Référence Spatiale (SRS), tels que Spherical-Mercator, sur demande. L’extrait du centre-ville plus petit est chargé dans une autre base de données PostGIS appelée « pgrouting ».
