pycsw¶

Catalogue de métadonnées¶

pycsw permet aux utilisateurs de publier un catalogue de descriptions de jeux de données (métadonnées) sur le web, en utilisant des interfaces et des formats basés sur des standards, de manière à ce qu’il soit interrogé et mis à jour par des clients de catalogage. pycsw est simple à configurer dans un serveur web Apache (ou n’importe quel serveur WSGI) et peut accéder à des enregistrements de métadonnées stockés comme des fichiers XML dans une base de données.

La métadonnée est la “donnée sur la donnée”, stockant des choses comme la date de création, l’auteur, le titre, la zone d’intérêt, etc, et est habituellement encodée dans des fichiers XML, en suivant les standards internationaux tels que ISO 19115, ISO 19139 et FGDC CSDGM.

pycsw est une implémentation en Python du Catalogue Service. Le projet est certifié OGC Compliant et est une OGC Reference Implementation. Le standard CSW définit les interfaces communes pour découvrir, naviguer et interroger des métadonnées à propos des données spatiales, des services web et des ressources associées.

pycsw est Open Source, mis à disposition sous licence MIT, et fonctionne sur toutes les plates-formes majeurs (Windows, Linux, Mac OS X).

Caractéristiques principales¶

  • Certifiée conforme à l’OGC et implémentations OGC de référence pour OGC CSW 2.0.2 et CSW 3.0.0
  • support du moissonnage pour WMS, WFS, WCS, WPS, WAF, CSW, SOS
  • implémente le Discovery Services 3.0 de INSPIRE
  • implémente ISO Metadata Application Profile 1.0.0
  • implémentation du profil d’application FGDC CSDGM Application pour CSW 2.0
  • Implémente le protocole de recherche Search/Retrieval par URL (SRU)
  • Implémente les fonctions de recherche Full Text
  • implémente les extensions OGC OpenSearch Geo/Time
  • implémente le protocole Open Archives Initiative pour le moissonnage de métadonnées
  • support des modèles de métadonnées ISO, Dublin Core, DIF, FGDC, Atom et GM03
  • déploiement CGI ou WSGI
  • Compatible Python 2 et 3
  • Configuration simple
  • Capacités transactionnelles (CSW-T)
  • Configuration de dépôt simple
  • Connectivité GeoNode
  • Connectivité HHypermap
  • Connectivité à “Open Data Catalog”
  • Connectivité CKAN
  • recherche distribuée de catalogue fédéré
  • validation de schéma XML temps-réel
  • architecture de plugins prévue pour être extensible
  • très léger (Python + CGI)

Standards implémentés¶

Standards OGC

  • Catalogue Service 2.0.2, 3.0.0
  • Filter Encoding (FE) 1.1.0, 2.0.0
  • OWS Common 1.0.0, 2.0.0
  • Geography Markup Language (GML) 3.1.1
  • Simple Feature Access (SFS) 1.2.1
  • OGC OpenSearch 1.0

Standards de métadonnées

  • Dublin Core 1.1
  • SOAP 1.2
  • ISO 19115:2003
  • ISO 19139:2007
  • ISO 19119:2005
  • NASA DIF 9.7
  • FGDC CSDGM 1998
  • GM03 2.1
  • SRU 1.1
  • OAI-PMH 2.0
  • A9 OpenSearch 1.1
