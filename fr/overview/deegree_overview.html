deegree¶

A propos¶

deegree fournit des services web géospatiaux basés sur les standards Open Geospatial Consortium (OGC) et les spécifications INSPIRE Network Services. deegree prend en charge:

  • Web Feature Service (WFS): Sert les données vectorielles à partir de données géospatiales

  • Web Map Service (WMS): Sert les cartes rendues à partir de données géospatiales

  • OpenGIS Web Map Tile Service (WMTS): Sert des tuiles de carte pré-calculées

  • Catalogue Service for the Web (CSW): Effectue des recherches de jeux de données et services géospatiaux

  • Web Processing Service (WPS): Exécute des processus de géotraitement

Caractéristiques principales¶

WFS

  • Implémente les standards WFS 1.0.0, 1.1.0 et 2.0.0

  • Official OGC reference implementation for WFS 1.1.0 (Basic and Transactional) and WFS 2.0.0 (Basic)

  • Complètement transactionnel (même pour les modèles de données riches)

  • Supporte des requêtes de type KVP, XML et SOAP

  • Entrées/sorties GML 2/3.0/3.1/3.2

  • Support des requêtes GetGmlObject et XLinks

  • Hautes performances et scalabilité excellente

  • Transformation de coordonnées à la volée

  • Conçu dès le départ pour des modèles de données riches

  • Backend supporte un mapping flexible des applications GML avec des modèles relationnels

  • Modèle géométrique conforme à la norme ISO 19107: géométries complexes (par ex. courbes non linéaires)

  • Prise en charge avancée de l’expression des filtres basée sur XPath 1.0

  • Prend en charge de nombreux backends, tels que PostGIS, Oracle Spatial, MS SQL Server, Shapefiles ou documents d’instance GML.

WMS

  • Implémente les standards WMS 1.1.1 et 1.3.0

  • Official OGC reference implementation for WMS 1.1.1 and WMS 1.3.0

  • Prise en charge étendue des langages de style SLD/SE versions 1.0.0 et 1.1.0

  • Hautes performances et scalabilité excellente

  • Rendu haute qualité

  • Style en fonction de l’échelle

  • Le support de SE supprime le besoin d’un grand nombre d’extensions propriétaires

  • Configuration facile du format HTML et d’autres formats de sortie pour les réponses GetFeatureInfo

  • Utilise l’accès aux flux de données avec empreinte mémoire minimale

  • Prise en charge presque complète de la symbolisation raster telle que définie par SE (avec quelques extensions)

  • Prise en charge complète de TIME/ELEVATION et d’autres dimensions pour les objets géographiques et les données raster

  • Supporte de nombreux backends, tels que PostGIS, Oracle Spatial, Shapefiles ou des documents d’instance GML.

  • Peut faire des rendus de modèles de données riches directement

WMTS

  • Implémente le standard basique WMTS 1.0.0 (KVP)

  • Official OGC reference implementation for WMS 1.0.0

  • Hautes performances et scalabilité excellente

  • Supporte différents backends, tels que les GeoTIFF, les WMS à distance ou les hiérarchies d’images de tuiles de système de fichiers.

  • Supporte la mise en cache à la volée (en utilisant EHCache)

  • Supporte GetFeatureInfo pour des backends WMS distants

CSW

  • Implémente le standard CSW 2.0.2

  • Complètement transactionnel

  • Supporte des requêtes de type KVP, XML et SOAP

  • Hautes performances et scalabilité excellente

  • Profil d’application des métadonnées ISO 1.0.0

  • La couche d’accès aux données enfichable et modulaire permet d’ajouter le support de nouveaux AP et backends.

  • L’architecture modulaire de l’inspecteur permet de valider les enregistrements à insérer en fonction de différents critères

  • Inspecteurs de standards: validité du schéma, intégrité de l’identificateur, exigences INSPIRE.

  • Gère toutes les propriétés interrogeables définies (pour Dublin Core ainsi que le profil ISO)

  • Expressions de filtres complexes

WPS

  • Implémente le standard WPS 1.0.0

  • Supporte des requêtes de type KVP, XML et SOAP

  • Couche connectable pour fournisseur de traitement

  • API facile à utiliser pour implémenter des traitements Java

  • Prise en charge de toutes les variantes des paramètres d’entrée/sortie: littérale, bbox, complexe (binaire et xml)

  • Accès par flux pour les paramètres d’entrée/sortie complexes

  • Traitement d’énormes quantités de données avec une empreinte mémoire minimale

  • Supporte le stockage des documents de réponse/paramètres de sortie

  • Supporte les paramètres d’entrée fournis “inline” et par référence

  • Supporte des réponses RawDataOutput/ResponseDocument

  • supporte l’exécution asynchrone (avec interrogation du statut de traitement)

Standards implémentés¶

  • Web Feature Service (WFS) 1.0.0, 1.1.0 et 2.0.0

  • Web Map Service (WMS) 1.1.1 et 1.3.0

  • OpenGIS Web Map Tile Service (WMTS) 1.1.0

  • Catalogue Service for the Web (CSW) 2.0.2

  • Web Processing Service (WPS) 1.0.0

  • Geography Markup Language (GML) 2.12, 3.0.1, 3.1.1 et 3.2.1
