EOxServer¶

A proposito di¶

EOxServer è un server per visualizzare dati e matadati Earth Observation (EO) attraverso Standard aperti.

I dati Earth Observation attualmente supportati includono raster 2D generalmente acquisiti usando sensori di satelliti o aerei includendo informazioni sulla data di acquisizione e il footprint del pianeta. È tipicamente usato per monitorare l’ambiente naturale e artificiale

EOxServer builds upon an Open Source stack of software which includes Python, MapServer, Django/GeoDjango, GDAL, PROJ, and a SpatiaLite or PostGIS database.

Caratteristiche principali¶

  • Visualizzazione, filtro, subset e scaricamento EO data
  • Registrazione di set di dati EO
  • Interfaccia di amministrazione sul web e da linea di comando
  • Publish data via Open Geospatial Consortium (OGC)
    • Web Coverage Service (WCS) 1.0, 1.1 and 2.0 & proposed EO-WCS
    • Web Map Service (WMS) & EO-WMS
    • Proposta estensione al WCS per includere: codifica GeoTIFF, proiezioni predefinite (CRSs), riscalamento, e interpolazione.
    • Protocolli supportati sono KVP, XML/POST (usato insieme con SOAP2POST Proxy per fornire un’interfaccia XML/SOAP)
  • Register data stored in S3
  • Pubblicazione usando i seguenti formati e gruppi:
    • 2-D EO Coverages derivato da gmlcov:RectifiedGridCoverage
    • 2-D EO Coverages derivato da gmlcov:ReferenceableGridCoverage
    • Serie di dataset come una collezione di EO Coverages es. in una serie temporale
    • Stitched Mosaics of Rectified EO Coverages including contributingFootprint
  • Formati di covarage supportati:
  • GeoTIFF
  • Formati supportati dalla libreria GDAL

Standards implementati¶

  • WCS, EO-WCS
  • WMS, EO-WMS
  • WPS
  • GML, GMLCOV, EO-O&M
  • OpenSearch
