MapCache¶

Servizio Web¶

MapCache è un server che implementa le cache di tile per accelerare l’accesso ai livelli WMS. Gli obiettivi primari sono di essere veloce e facilmente implementabile, mentre offre le caratteristiche essenziali (e molto di più!) che ci si aspetta da una soluzione di cache di tile.

MapCache fa parte della famiglia di MapServer, ma può essere configurato per mettere in cache e servire dati da qualsiasi server geospaziale che usi WMS. MapCache può essere eseguito su Apache come modulo o su qualsiasi server web che supporti CGI/FastCGI come IIS, Nginx e Jetty.

Caratteristiche principali¶

  • Permette la cache delle tile di essere salvate in

    • su disco

    • sul cloud utilizzando REST (S3, Azure, Google)

    • in database inclusi SQLite, Berkeley DB, Riak

    • in Memcache

  • Include un’applicazione di seeder

  • Cache basata su dimensioni (come il tempo)

  • Permette la cache di Mapbox Vector (MVT) e tile UTFGrid

  • Griglie personalizzabili

  • Permette di «assemblare» le tile insieme - permettendo alle tile di diversi set di dati di essere fuse insieme in una singola tile da inviare al client

  • Include un’applicazione seeder per precompilare le cache dalla linea di comando

  • Permette alle richieste di essere reindirizzate a un server

  • Un’interfaccia demo integrata

Standard implementati¶

  • Supporto di numerosi standard Open Geospatial Consortium (OGC)

    • Web Map Service (WMS)

    • OpenGIS Web Map Tile Service (WMTS)

  • Standard aggiuntivi includono:

    • TMS

    • Google Maps XYZ

    • KML SuperOverlays
