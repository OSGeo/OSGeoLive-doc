deegree¶

A proposito di¶

degree fornisce servizi web geospaziali basati sulle specifiche Open Geospatial Consortium (OGC) e INSPIRE Network Services. degree supporta:

  • Web Feature Service (WFS): Fornisce dati vettoriali da dati geospaziali

  • Web Map Service (WMS): Fornisce mappe disegnate da dati geospaziali

  • OpenGIS Web Map Tile Service (WMTS): Fornisce mappe tassellate già disegnate

  • Catalogue Service for the Web (CSW): Esegue ricerce per trovare dati e servizi geospaziali

  • Web Processing Service (WPS): Esegue processi geospaziali

Caratteristiche principali¶

WFS

  • Implementa standard WFS 1.0.0, 1.1.0 e 2.0.0

  • Implementazione di riferimento ufficiale OGC per WFS 1.1.0 (Base e Transazionale) e WFS 2.0.0 (Base)

  • Completamente transazionale (anche per modelli di dati ricchi)

  • Supporta richieste KVP, XML e SOAP

  • GML 2/3.0/3.1/3.2 output/input

  • Supporto per richieste GetGmlObject e XLinks

  • Performance elevate ed eccellente scalabilità

  • Trasformazione di coordinate al volo

  • Progettato per modelli di dati ricchi dal basso verso l’alto

  • I backend supportano la mappatura flessibile degli schemi applicativi GML ai modelli relazionali

  • Modello di geometria conforme a ISO 19107: Geometrie complesse (per esempio curve non lineari)

  • Supporto a espressioni avanzate per i filtri basate su XPath 1.0

  • Supporta numerosi backend, come PostGIS, Oracle Spatial, MS SQL Server, Shapefiles o documenti di istanza GML

WMS

  • Implementa standards WMS 1.1.1 e 1.3.0

  • Official OGC reference implementation for WMS 1.1.1 and WMS 1.3.0

  • Supporto esteso per i linguaggi di stile SLD/SE versioni 1.0.0 e 1.1.0

  • Performance elevate ed eccellente scalabilità

  • Rendering di alta qualità

  • Stili in funzione della scala

  • Il supporto per SE rimuove la necessità di molte estensioni proprietarie

  • Facile configurazione di HTML e altri formati di output per le risposte GetFeatureInfo

  • Utilizza un accesso ai dati stream-based con un ingombro minimo della memoria

  • Supporto quasi completo per la simbolizzazione raster come definita in SE (con alcune estensioni)

  • Supporto completo per TEMPO/ALTEZZA e altre dimensioni sia per i vettoriali che per i dati raster

  • Supporta numerosi backend, come PostGIS, Oracle Spatial, Shapefiles o documenti di istanza GML

  • Può renderizzare direttamente modelli di dati complessi

WMTS

  • Implementa lo standard WMTS base 1.0.0 (KVP)

  • Implementazione ufficiale di riferimento OGC per WMS 1.0.0

  • Performance elevate ed eccellente scalabilità

  • Supporta diversi backend, come GeoTIFF, WMS remoti o gerarchie di immagini tile dal file system

  • Supporta cache al volo (usando EHCache)

  • Supporta GetFeatureInfo per backend WMS remoti

CSW

  • Implementa standard CSW 2.0.2

  • Completamente transazionale

  • Supporta richieste KVP, XML e SOAP

  • Performance elevate ed eccellente scalabilità

  • ISO Metadata Application Profile 1.0.0

  • Il livello di accesso ai dati pluggable e modulare permette di aggiungere il supporto per nuovi AP e backend

  • L’architettura modulare dell’inspector permette di convalidare i record da inserire secondo vari criteri

  • Ispettori standard: validità dello schema, integrità degli identificatori, requisiti INSPIRE

  • Gestisce tutte le proprietà interrogabili definite (per Dublin Core e per il profilo ISO)

  • Espressioni filtro complesse

WPS

  • Implementa standard WPS 1.0.0

  • Supporta richieste KVP, XML e SOAP

  • Layer del fornitore di processi collegabile

  • API facile da usare per implementare processi Java

  • Supporta tutte le varianti di parametri di input/output: letterale, bbox, complesso (binario e xml)

  • Accesso in streaming per parametri di input/output complessi

  • Elaborazione di enormi quantità di dati con un ingombro minimo di memoria

  • Supporta la memorizzazione di documenti di risposta/parametri di output

  • Supporta parametri di input dati in linea e per riferimento

  • Supporta le risposte RawDataOutput/ResponseDocument

  • Supporta l’esecuzione asincrona (con polling dello stato del processo)

Standard implementati¶

  • Web Feature Service (WFS) 1.0.0, 1.1.0 e 2.0.0

  • Web Map Service (WMS) 1.1.1 e 1.3.0

  • OpenGIS Web Map Tile Service (WMTS) 1.1.0

  • Catalogue Service for the Web (CSW) 2.0.2

  • Web Processing Service (WPS) 1.0.0

  • Geography Markup Language (GML) 2.12, 3.0.1, 3.1.1 e 3.2.1
