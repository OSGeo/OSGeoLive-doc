Guida Rapida 52 North WPS¶

Il 52°North WPS è un framework per l’elaborazione basata sul web di dati (geospaziali). Il servizio implementa lo standard di interfaccia OGC Web Processing Service 1.0.

Questa guida rapida descrive come elaborare i dati e memorizzare il risultato in un altro servizio web utilizzando le operazioni WPS.

  • Ottenere le informazioni sul servizio

Avvia 52°North WPS¶

Per lanciare il WPS di 52°North WPS sul Live DVD, vai su Geospatial ‣ Web Services ‣ 52North ‣ Start 52North WPS o usa questo link diretto.

Apparirà la pagina di benvenuto di 52n WPS.

(Se il servizio web non è raggiungibile, prova ad avviare il motore servlet Tomcat seguendo i passi in fondo al Pagina di avvio rapido di SOS.)

Ottenere le informazioni sul servizio¶

Controlla le capabilities del WPS 52°North con questa richiesta:

http://localhost:8080/52nWPS/WebProcessingService?Request=GetCapabilities&Service=WPS

Process data¶

  1. Clicca sul link per aprire il 52nWPS-TestClient.

  2. Assicurati che GeoServer sia in esecuzione poiché la dimostrazione richiede dati dall’installazione locale di GeoServer.

  3. Per farlo, prova ad aprire la pagina http://localhost:8082/geoserver/web. Se la pagina non è stata trovata, seleziona Geospatial ‣ Web Services ‣ GeoServer ‣ Start GeoServer nel menu. Il GeoServer dovrebbe essere in esecuzione dopo pochi istanti.

  4. Seleziona la richiesta di esempio «SimpleBuffer_out_wfs.xml» dall’elenco a discesa:

  5. Premi il pulsante Invia e la richiesta sarà trasmessa al WPS 52°North che genererà un buffer intorno alle strade principali della Tasmania con una larghezza di 0,05 gradi e memorizzerà il risultato nel GeoServer.

  6. Copia il ResourceID dalla risposta. Questo è il nome del layer GeoServer. Aggiungi l’ID alla seguente richiesta: http://localhost:8082/geoserver/wms?service=WMS&version=1.1.0&request=GetMap&styles=&bbox=145.14757902405984,-43.47330297262748,148.32274986232298,-40.80286290459129&width=512&height=430&srs=EPSG:4326&format=application/openlayers&layers=Add-ResourceID-here Dovresti ottenere il seguente risultato:

Cose da provare¶

  • Prova altre richieste di esempio dall’elenco a discesa del client di prova WPS.

  • Potresti anche installare il plugin WPS per QGIS e provarlo con il WPS 52°North.

E ora?¶

  • Crea i processi ed eseguili usando vari client.

  • Imposta il WPS per eseguire i processi di GRASS GIS 7.

  • Imposta il WPS per eseguire gli script R come processi.
