MapCache¶

Webbtjänst¶

MapCache är en server som implementerar cachelagring av plattor för att snabba upp åtkomsten till WMS-lager. De primära målen är att vara snabb och lätt att distribuera, samtidigt som den erbjuder de viktigaste funktionerna (och mer!) som förväntas av en lösning för cachelagring av plattor.

MapCache är en del av MapServer-familjen, men kan konfigureras för att cachelagra och servera data från alla geospatiala servrar som använder WMS. MapCache kan köras på Apache som en modul eller på vilken webbserver som helst som stöder CGI/FastCGI, t.ex. IIS, Nginx och Jetty.

Kärnfunktioner¶

  • Tillåter att cachelagring av plattor lagras i

    • på disk

    • i molnet med hjälp av REST (S3, Azure, Google)

    • i databaser inklusive SQLite, Berkeley DB, Riak

    • i Memcache

  • Inkluderar en såmaskinstillämpning

  • Cachelagring baserad på dimensioner (t.ex. tid)

  • Tillåter cachelagring av Mapbox Vector (MVT) och UTFGrid-brickor

  • Anpassningsbara rutnät

  • Gör det möjligt att ”montera” ihop plattor - så att plattor för olika dataset kan slås ihop till en enda platta som skickas till klienten

  • Inkluderar en seeder-applikation för att förfylla cacheminnen från kommandoraden

  • Tillåter att förfrågningar proxas tillbaka till en server

  • Ett inbyggt demogränssnitt

Implementerade standarder¶

  • Stöd för ett stort antal standarder från Open Geospatial Consortium (OGC)

    • Web Map Service (WMS)

    • OpenGIS Web Map Tile Service (WMTS)

  • Ytterligare standarder inkluderar:

    • TMS

    • Google Maps XYZ

    • KML SuperOverlays
