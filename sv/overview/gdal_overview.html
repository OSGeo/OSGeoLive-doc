GDAL/OGR¶

Om¶

Geospatial Data Abstraction Library (GDAL/OGR) tillhandahåller kommandoradsverktyg för att översätta och bearbeta ett brett spektrum av raster- och vektorformat för geospatiala data.

Verktygen är baserade på ett plattformsoberoende C++-bibliotek som är tillgängligt via många olika programmeringsspråk. Som ett bibliotek presenterar det en enda abstrakt datamodell för den anropande applikationen för alla format som stöds.

GDAL/OGR är det mest använda biblioteket för åtkomst till geospatiala data. Det utgör den primära datatillgångsmotorn för många applikationer, inklusive MapServer, GRASS, QGIS och OpenEV. Det används också av paket som OSSIM, Cadcorp SIS, FME, Google Earth, VTP, Thuban, ILWIS, MapGuide och ArcGIS.

Kärnfunktioner¶

  • Kommandoradsverktyg för dataöversättning, bildförvrängning, underindelning och många andra vanliga uppgifter

  • Mycket effektiv åtkomst till rasterdata, med utnyttjande av tiling och översikter

  • Stöd för stora filer - större än 4 GB

  • Library access from Python, Java, and C#

  • Coordinate system engine built on PROJ and OGC Well Known Text coordinate system descriptions

Populära format som stöds¶

GDAL stöder över 140 rasterformat och OGR över 80 vektorformat.

Dessa inkluderar:

  • Raster: GeoTIFF / COG, HDF, NetCDF, GeoPackage, JPEG2000, ECW, ENVI, KEA, MrSID, DTED, NITF and more …

  • Vector: GeoPackage, ESRI Shapefile, GeoJSON, PostGIS, (Geo)Parquet, ESRI File Geodatabase, FlatGeobuf, GML, AutoCAD DWG, MapInfo, DGN, Oracle Spatial, and more …

Implementerade standarder¶

OGR:s vektordatamodell är förenlig med OGC:s specifikation för enkla funktioner.
