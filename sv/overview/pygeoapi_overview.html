pygeoapi¶

Webbtjänst¶

pygeoapi är en Python-serverimplementering av OGC API-sviten med standarder. Projektet uppstod som en del av nästa generations OGC API-insatser 2018 och ger organisationer möjlighet att distribuera en RESTful OGC API-slutpunkt med OpenAPI, GeoJSON och HTML. pygeoapi är öppen källkod och släpps under en MIT-licens.

Kärnfunktioner¶

  • en modern OGC API-server som levereras direkt ur lådan

  • certifierad OGC-kompatibel och referensimplementering

  • implementerar dessutom OGC API - Features, OGC API - Coverages, OGC API - Tiles, OGC API - Processes, OGC API - Records, OGC API - Environmental Data Retrieval och SpatioTemporal Asset Library

  • out of the box dataleverantör plugins för rasterio, GDAL /OGR, Elasticsearch, PostgreSQL / PostGIS

  • lättanvänd OpenAPI / Swagger-dokumentation för utvecklare

  • stödjer JSON-, GeoJSON-, HTML- och CSV-utdata

  • stöder datafiltrering genom spatiala, tidsmässiga eller attributfrågor

  • enkel att installera: installera en fullständig implementation via pip eller git

  • enkel YAML-konfiguration

  • lätt att driftsätta: via UbuntuGIS eller den officiella Docker-avbildningen

  • flexibel: bygger på ett robust plugin-ramverk för att skapa anpassade datakopplingar, format och processer

  • stöder alla Python-webbramverk (ingår är Flask [standard], Starlette)

Implementerade standarder¶

Standarder är kärnan i pygeoapi. Nedan visas projektets matris för stöd av standarder.

  • Implementering: implementerar standard (bra)

  • Compliant: uppfyller OGC:s krav på efterlevnad (bra)

  • Referensimplementering: ger en referens för standarden (fantastiskt!)
