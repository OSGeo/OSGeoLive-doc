EOxServer¶

Om¶

EOxServer är en server för presentation av stora dataarkiv och metadata för jordobservation (EO) via öppna standarder.

Jordobservationsdata som för närvarande stöds inkluderar 2D-rasterdata som vanligtvis förvärvas med hjälp av sensorer på satelliter eller flygplan och inkluderar information om förvärvstidpunkten och fotavtrycket på planeten Jorden. Det används vanligtvis för att övervaka jordens naturliga och byggda miljö.

EOxServer bygger på en Open Source-programvarustack som innehåller Python, MapServer, Django/GeoDjango, GDAL, PROJ och en SpatiaLite eller PostGIS databas.

Kärnfunktioner¶

  • Visa, filtrera, dela upp och ladda ner EO-data

  • Registrera EO:s dataarkiv

  • Webbaserat administrationsgränssnitt och kommandoradsgränssnitt

  • Publicera data via Open Geospatial Consortium (OGC)

    • Web Coverage Service (WCS) 1.0, 1.1 och 2.0 & föreslagen EO-WCS

    • Web Map Service (WMS) & EO-WMS

    • Föreslagna tillägg till WCS inklusive: GeoTIFF-kodning, fördefinierade projektioner (CRS), skalning och interpolering.

    • Protokoll som stöds är: KVP och XML/POST (används tillsammans med SOAP2POST Proxy för att även tillhandahålla XML/SOAP)

  • Registrera data som lagrats i S3

  • Publicera med hjälp av följande format och grupperingar:

    • 2-D EO-täckningar härledda från gmlcov:RectifiedGridCoverage

    • 2-D EO-täckningar härledda från gmlcov:ReferenceableGridCoverage

    • Dataset Series som en samling av EO-täckningar, t.ex. i en tidsserie

    • Stitched Mosaics av rektifierade EO-täckningar inklusive bidragande fotavtryck

  • Stöd för täckningsformat:

  • GeoTIFF

  • Format som stöds av GDAL-biblioteket

Implementerade standarder¶

  • WCS, EO-WCS

  • WMS, EO-WMS

  • WPS

  • GML, GMLCOV, EO-O&M

  • OpenSearch
