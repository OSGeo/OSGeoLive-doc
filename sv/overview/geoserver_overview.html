GeoServer¶

Webbtjänst¶

GeoServer är en webbserver som gör det möjligt att visa kartor och data från en mängd olika format till standardklienter som webbläsare och GIS-program. Data publiceras via standardbaserade gränssnitt, t.ex. WMS, WFS, WCS, WPS, Tile Caching med mera. GeoServer levereras med ett webbläsarbaserat hanteringsgränssnitt och ansluter till flera datakällor i backend.

Kärnfunktioner¶

Servera data från en mängd olika datalager:
  • Vektor

    • Shapefiler, extern WFS

    • PostGIS, ArcSDE, DB2, Oracle Spatial, MySql, SQL Server

  • Raster

    • GeoTiff, JPG och PNG (med världsfil), bildpyramid, GDAL-format, Image Mosaic, Oracle GeoRaster, NetCDF

Data serveras som säkra och snabba bilder med hjälp av WMS- och WMTS-protokoll:

Eftersom uppgifterna serveras som en bild är dina uppgifter helt trygga och säkra. Om de inte redigerar dina data finns det inget sätt att stjäla dina data. Utseendet på varje kartlager kan kontrolleras med hjälp av SLD-standarden som gör att funktioner kan färgas och märkas. Genom att kombinera dessa regler med OGC Filters kan skalberoende styling implementeras, vilket gör att fler detaljer kan läggas till när en användare zoomar in. Kollisionshantering, gruppering och prioritering av etiketter implementeras också.

Fullständig vektordata kan skickas till en klient med hjälp av WFS-protokollet:

En WFS-klient kan ladda ner vektordata och använda dem för kartläggning, spatial analys och andra operationer. Om användaren är behörig kan han eller hon också ändra data och skicka tillbaka dem till servern för att uppdatera de lagrade data med hjälp av WFS-T-protokollet. Data kan överföras med hjälp av GML (komprimerat) samt andra standarddataformat som shapefile och json.

Rasterdatavärden kan skickas till en klient med hjälp av WCS-protokollet:

En GIS-klient kan begära de faktiska rasterdata för användning i spatial analys. Detta gör det möjligt för en användare att skapa applikationer som kan modellera den process som beskrivs av dina data.

Reprojektering i farten:

GeoServer stöder majoriteten av EPSG:s databas med projektioner och kan på begäran reprojicera till vilken som helst av dem, så att klienter med begränsat stöd för reprojektion kan överföra belastningen till servern.

Täckning Cache

GeoWebCache, som är integrerad i GeoServer, kaklar och cachelagrar kartbilder från en mängd olika källor, t.ex. WMS. Den implementerar olika tjänstegränssnitt (t.ex. WMS-C, WMTS, TMS, Google Maps KML, Virtual Earth) för att påskynda och optimera leverans av kartbilder. Den kan också kombinera om brickor för att fungera med vanliga WMS-klienter.

Web Processing Service (WPS)

Geospatiala algoritmer, inklusive de från JTS, kan anropas via GeoServers gränssnitt Web Processing Service (WPS). Detta inkluderar kedjning av processer och användning av renderingstransformationer för att skapa kraftfulla visualiseringar via WMS-gränssnittet.

Implementerade standarder¶

Stöd för många standarder från Open Geospatial Consortium (OGC):

  • Web Map Service (WMS) (klient/server)

  • OpenGIS Web Map Tile Service (WMTS)

  • Web Feature Service (WFS) (transaktionell)

  • Web Coverage Service (WCS)

  • Filter Encoding (FE)

  • Styled Layer Descriptor (SLD)

  • Geography Markup Language (GML)

  • KML
