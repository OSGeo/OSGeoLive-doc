MapProxy クイックスタート¶

MapProxyはすべてのWMSとタイルサービス用の いわば スイスアーミーナイフです。既存のマップサービスのデータをキャッシュし、高速化し、変換して、任意のデスクトップまたはWeb GISクライアントに提供します。

MapProxyはタイルキャッシュソリューションであるだけでなく、WMSクライアントの完全サポートなど、多くの新しい革新的な機能を提供します。

サービスの開始とデモの表示¶

MapProxyには柔軟な配置オプションがあり、例えばApacheに統合することができます。さらに、スタンドアロンサービスとしてMapProxyを開始することもできます。

  1. MapProxy をスタンドアロンサービスとして起動するには、 Geospatial ‣ Web Services ‣ MapProxy ‣ Start MapProxy をクリックします。
  2. Firefoxを開き、http://localhost:8011/demo に移動します

MapProxyのデモページには、基本的なWMSと各設定レイヤのタイルクライアントが含まれています。たとえば、WMSレイヤの投影選択ができませんが、インストール後に動作確認するには十分です。

デスクトップGISで表示する¶

uDig、QGIS、gvSIGなど、任意のWMS Desktop GISにMapProxyを追加できます。サービス URL: http://localhost:8011/service?

MapProxyの設定例には、異なるマップエンジンとマップデータを使用するレイヤがいくつか含まれています。各レイヤの詳細については、以下を参照してください。

2つのカスケードソースからのデータと、1つのソースからの FeatureInformation による単一のWMSレイヤ

使用可能なレイヤ¶

mapnik:
Mapnikが描いた世界人口。Mapnikサービスを開始する必要はありません。なぜなら、このレイヤはMapProxyで統合されたMapnikサポートを利用しているからです。
tilestache:
TileStacheは、Mapnik世界人口サービスからタイルを配信するサーバーです。このレイヤは、MapProxyの既存のタイルサービスを組み込み、WMSとして使用できるようにする機能を示しています。このレイヤにアクセスする前に、 Start Mapnik & TileStache でTileStacheを開始する必要があります。
geoserver:
GeoServer WMSによるUSの人口統計のレンダリングこのレイヤは、ソースWMSにカスケードされる GetFeatureInfo 要求をサポートします。このレイヤにアクセスする前に、 Geospatial ‣ Web Services ‣ GeoServer ‣ Start GeoServer でGeoServerを起動する必要があります。
mapnik_geoserver:
このレイヤは、複数のソースを1つのレイヤに結合する機能を示しています。Mapnikの世界人口とGeoServerの米国人口を組み合わせたものです。USの人口統計についての GetFeatureInfo サポートは引き続き利用可能です。このレイヤにアクセスする前に、 Geospatial ‣ Web Services ‣ GeoServer ‣ Start GeoServer でGeoServerを起動する必要があります。
mapserver:
このレイヤはMapserver Itascaデモデータセットを使用します。ローカルデータセットであり、 カバレッジ 機能を使用してBBOXをItasca,ILに制限します。

独自の構成を作成する¶

mapproxy-util を使って新しい設定テンプレートを作成し、テストサーバを起動することができます。

新規構成を作成する手順は、次のとおりです:

mapproxy-util create -t base-config ~/mapproxy

作成した設定でポート8011上のテストサーバを起動するには、次の手順を実行します:

mapproxy-util serve-develop ~/mapproxy/mapproxy.yaml -b 0.0.0.0:8011

http://127.0.0.1:8011/demo/ でデモページを見ることができます。設定ファイルを変更すると、MapProxyが自動的にリロードされます。
