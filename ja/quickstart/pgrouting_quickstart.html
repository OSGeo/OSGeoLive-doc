pgRouting クイックスタート¶

pgRouting は、経路探索と他のネットワーク解析機能を PostGIS/PostgreSQL データベースに追加する拡張機能です。

目次

  • データベース内で pgRouting を有効化

  • osm2pgrouting での OSM データの読み込み

  • インポートしたデータをチェックします

  • 内部クエリ

  • pgr_Dijkstra

  • 次のステップ

データベース内で pgRouting を有効化¶

この例では、 city_routing というデータベースを作成し、そのデータベースで pgRouting を有効化します。

  • システムツール ‣ QTerminal ウィンドウを開き、PostgreSQL の psql コマンドラインツールを実行します。

psql

psql プロンプトで次のように入力します:

CREATE DATABASE city_routing;
\connect city_routing;
CREATE EXTENSION pgrouting CASCADE;

インストールの確認は、以下の実行により可能です:

SELECT * FROM pgr_version();

pgr_version
-------------
 3.1.0
(1 row)

バージョンは 3.1.0 かそれ以上である必要があります。

データベースから抜けます

\q

osm2pgrouting での OSM データの読み込み¶

osm2pgrouting は、pgRouting互換形式でデータベースに .osm ファイルをロードするためのコマンドラインツールです。

インストールされているバージョンを確認します

osm2pgrouting --version

出力が表示されます

This is osm2pgrouting Version 2.3.6

osmファイルからデータをロードします:

bzcat data/osm/feature_city.osm.bz2 > /tmp/feature_city.osm
osm2pgrouting \
  -f /tmp/feature_city.osm \
  -h localhost \
  -U user \
  -d city_routing \
  -p 5432 \
  -W user \
  --conf=/usr/share/osm2pgrouting/mapconfig_for_cars.xml
rm /tmp/feature_city.osm

osm2pgrouting は OSM 識別子 osm_id をロードし、すべてのデータに対して一意の識別子を生成します: 頂点に対しては id、エッジに対しては gid となります。

SELECT id, osm_id
FROM ways_vertices_pgr
WHERE id IN(100,600);

結果は次のとおりです:

-----+----------

 600 | 82708785

内部クエリ¶

ほとんどの pgRouting 関数には SQL ステートメントであるパラメータがあり、これは 内部クエリ と呼ばれます。

内部 SQL ステートメントは常にフィールド名 id、source、target および cost を持たなければならず、reverse_cost はオプションとして扱われます。

セグメントの識別子として gid を使用する内部クエリ

SELECT gid as id,
       source, target,
       cost, reverse_cost
FROM ways

セグメントの識別子として gid、長さを cost として使用する、オプションの reverse_cost なしの内部クエリ

SELECT gid as id,
       source, target,
       length AS cost
FROM ways

pgr_Dijkstra¶

ダイクストラ法による最短経路探索関数を、無向グラフ上のセグメントを横断する時間(秒)に基づき、頂点の識別子として id を使用して実行します

SELECT *
FROM pgr_dijkstra(
  'SELECT gid as id,
          source, target,
          cost_s AS cost, reverse_cost_s AS reverse_cost
  FROM ways',
  100, 600,
  directed => false
);

SELECT *
FROM pgr_dijkstra(
  'SELECT gid as id,
          source_osm AS source, target_osm AS target,
          cost_s AS cost, reverse_cost_s AS reverse_cost
  FROM ways',
  81622364, 82708785,
  directed => false
);

クエリはフィールド source_osm と target_osm を使用しているので、クエリは source と target という必要な名前となるよう、別名を付けます。

経路のジオメトリを取得するクエリ

pgr_dijkstra の結果は、ways テーブルに結合する必要があります。

SELECT seq, edge, rpad(b.the_geom::text,60,' ') AS "the_geom (truncated)"
FROM pgr_dijkstra(
  'SELECT gid as id,
          source, target,
          cost_s AS cost, reverse_cost_s AS reverse_cost
  FROM ways',
  100, 600,
  directed => false
) AS a
JOIN ways AS b ON (a.edge = b.gid) ORDER BY seq;
