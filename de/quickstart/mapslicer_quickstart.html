MapSlicer Quickstart¶

Mit MapSlicer kann man Kacheln ausgehend von georeferenzierten Kartenbildern erzeugen. Die Kacheln können dann über einen Webserver als georeferenzierte Karte eingebunden und präsentiert werden. MapSlicer folgt dabei der Tile Map Service (TMS) Spezifikation der Open Source Geospatial Foundation (OSGeo).

  • Eingangsparameter: Beliebige georeferenzierte Bilddateien, z.B. TIFF, JPEG von ArcGIS, QGIS.
  • Ausgabeoptionen: Verzeichnisse mit gekachelten Bildern, die in einen Webserver kopiert werden können.
  • Stärken: MapSlicer is einfach, frei, Open-Source Software mit einem mit einem einfachen Assistenten, der untern beschrieben und auf die Optionen eingegangen wird.

MapSlicer nutzt den Vorteil, dass Google Maps , Microsoft Bing , Yahoo Maps und andere Online-Kartenprovider, inklusive OpenStreetMap , die gleiche Kartenprojektion und das Kachelschema verwenden. Somit sind die Kacheln kompatibel. MapSlicer bereitet die Kacheln soweit vor, dass diese zur Spezfikation passen und ist somit ein einfaches Tool, um georeferenzierte Karten in einer Mashup Webseite einzubinden.

MapSlicer starten¶

  1. Download MapSlicer oder starten Sie das Programm direkt von der OSGeo-Live Distribution.

  2. Wählen Sie das Tile Profile (Kachelprofil) aus. Wählen Sie Image Based Tiles (raster) für die Standard-Webveröffentlichung. Klicken Sie auf Weiter.

  3. Wählen Sie die Source Data Files (Daten). Navigieren Sie, um das Rasterbild auszuwählen, das Sie kacheln möchten. Es ist auch möglich, eine NODATA-Farbe auszuwählen, die im resultierenden Bild als transparent erscheint. Klicken Sie auf Weiter.

  4. Geben Sie das Räumliche Referenzsystem / Koordinatensystem des Bildes an. Geben Sie die ID-Nummer aus der EPSG-Datenbank an. Klicken Sie auf Weiter.

  5. Geben Sie Details zur Kachelpyramide an. Die Standardeinstellungen für Zoomstufen und Dateiformat sind oft am besten. Klicken Sie auf Weiter.

  6. Geben Sie Details zum Zielordner und Adressen /URLs für das Kachelset an. Wenn Sie diese nicht kennen, können sie nach der Kachelgenerierung in die Standarddateien googlemaps.html und openlayers.html eingefügt werden. Klicken Sie auf Weiter.

  7. Klicken Sie Viewers that should be generated. Standardmäßig wird eine openlayers.html-Datei generiert. Klicken Sie auf Weiter.

  8. Geben Sie die Details zur Erstellung des Viewers an, z. B. Titel, Copyright-Hinweis und API-Schlüssel. Wenn Sie diese nicht kennen, können sie nach der Kachelgenerierung in die Standarddateien googlemaps.html / openlayers.html eingefügt werden. Klicken Sie auf Weiter.

  9. Klicken Sie auf Render, um mit dem Rendern des Bildes zu beginnen.

  10. Nach Beendigung, stellt MapSlicer einen Link zum fertigen Kachelsatz bereit.

  1. Öffnen Sie die openlayers.html-Dateien in einem Webbrowser, um das Kachelset als Overlay auf der Live-Onlinekarte anzuzeigen. Um dies im Web zu präsentieren, kopieren Sie einfach das gesamte Kachelset und alle Unterverzeichnisse auf einen Webserver und bearbeiten Sie die Dateien googlemaps.html oder openlayers.html nach Bedarf.

Verwenden von gdal2tiles zum Erstellen einer MBTiles-Datenbank¶

MapSlicer ist im Wesentlichen ein GUI-Frontend zum Skript „gdal2tiles.py“. Eine aktualisierte Version von „gdal2tiles.py“ ist auch auf OSGeoLive vorhanden. Die daraus resultierenden Kachelbäume, die von dieser Software erstellt wurden, können mit dem Programm ‚mb-util‘ verwendet werden, um eine mbtiles-Datenbank zu erstellen, um Sie mit Smartphone-Software wie Geopaparazzi zu verwenden.

Um eine MBTiles-SQLite-Datenbank aus dem neu erstellten Kachelbaum zu erstellen, öffnen Sie ein Terminal,

cp data/raster/bluemarble.tif .
mapslicer    # (select /home/user/bluemarble.tif for input, JPEG as format)
mb-util --scheme=tms --image_format=jpg bluemarble bluemarble.mbtiles
qgis bluemarble.mbtiles

Weitere Informationen zum Ausführen dieser Werkzeuge finden Sie unter „gdal2tiles.py –help“ und „mb-util –help“.
