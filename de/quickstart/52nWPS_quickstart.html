52°North WPS Quickstart¶

52°North WPS ist ein Framework zur internetbasierten Prozessierung von (geospatialen) Daten. Der Dienst setzt den „OGC Web Processing Service 1.0 interface standard“ um.

Dieser Quickstart beschreibt die Prozessierung von Daten und die Speicherung der Ergebnisse in Internetdiensten mittels WPS Operationen.

Starten Sie den 52°North WPS¶

Um den 52°North WPS auf der Live-DVD auszuführen, gehen Sie zu Geospatial ‣ Web Services ‣ 52North ‣ Start 52North WPS oder verwenden Sie diesen ‚direkten Link <http: localhost:8080/52nwps/=““>‘_.</http:>

Die 52n WPS Willkommenseite wird erscheinen.

(Ist der Service nicht erreichbar, versuchen Sie den Tomcat Service zu starten. Folgen Sie dabei den einzelnen Schritten, wie es am Ende des Dokumentes SOS Quickstart beschrieben ist.)

Ausgabe von Information über den Dienst¶

Prüfen Sie das Capabilities-Dokument des 52°North WPS über den folgenden Aufruf:

http://localhost:8080/52nWPS/WebProcessingService?Request=GetCapabilities&Service=WPS

Prozessieren von Daten¶

  1. Klicken Sie auf den Link in der Sektion Test Client, um einen Test-Client zu öffnen.

  2. Stellen Sie sicher, dass der GeoServer gestartet ist, da die Demo Daten dieses lokalen Services verwendet.

  3. Sie können dies prüfen, indem Sie den folgenden Link aufrufen http://localhost:8082/geoserver/web. Wenn die Seite nicht angezeigt werden konnte, wählen Sie Geospatial ‣ Web Services ‣ GeoServer ‣ Start GeoServer im Menü. GeoServer sollte nach einigen Augenblicken starten.

  4. Wählen Sie den einfachen Aufruf „SimpleBuffer_out_wfs.xml“ aus der Auswahlliste:

  5. Drücken Sie den Send -Button und die Anfrage wird an den 52°North WPS Service übermittelt. Dieser wird eine XML-Repräsentation der Puffer um die Hauptstraßen von Tasmanien mit einem Abstand von 0.05 Grad zurückliefern und in GeoServer speichern.

  6. Kopieren Sie den Wert der ResourceID vom zurückgelieferten Dokument. Dies ist der Name des GeoServer Layers. Fügen Sie die ID zur folgender Anfrage hinzu: http://localhost:8082/geoserver/wms?service=WMS&version=1.1.0&request=GetMap&styles=&bbox=145.14757902405984,-43.47330297262748,148.32274986232298,-40.80286290459129&width=512&height=430&srs=EPSG:4326&format=application/openlayers&layers=Fügen-Sie-hier-die-ResourceID-ein Folgendes Ergebnis sollte zu sehen sein:

Was Sie noch ausprobieren können¶

  • Versuchen Sie weitere Besipielaufrufe aus der Auswahlliste des WPS Testclients.
  • Sie können das WPS Plugin für QGIS installieren und es mit dem 52°North WPS testen.

Was kommt als Nächstes?¶

  • Erstellen Sie eigene Prozesse und führen Sie diese in verschiedenen Clients aus.
  • Setzen Sie einen WPS zum Ausführen von GRASS GIS 7 Prozessen auf.
  • Setzen Sie einen WPS zum Ausführen von R Skripten auf.
