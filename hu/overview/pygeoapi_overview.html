pygeoapi¶

Web szolgáltatás¶

A pygeoapi egy Python szerver implementáció az OGC API szsvány sorozathoz. A projekt a következő generációs OGC API erőfeszítések részeként jelent meg 2018-ban és lehetőséget biztosít a szervezeteknek egy RESTful OGC API végpont felvonultatására OpenAPI, GeoJSON és HTML használatával. A pygeoapi nyílt forráskódú és MIT licenc alatt adják ki.

Alapfunkciók¶

  • kész modern OGC API szerver
  • tanúsított OGC-kompatibilis és referenciaimplementáció OGC API-elemekhez
  • továbbá implementálja az OGC API - feddvény, OGC API - csempék, OGC API - feldolgozás és a tér-idő eszközök könyvtárat
  • kész adatszolgáltató modulok a rasterio-hoz, GDAL/OGR-hez, Elasticsearch-hez, PostgreSQL/PostGIS-hez
  • könnyen használható OpenAPI / Swagger dokumentáció fejlesztőknek
  • támogatja a JSON, GeoJSON, HTML és CSV outputot
  • támogatja a térbeli, időbeli vagy attribútum lekérdezés szerinti adatszűrést
  • könnyen telepíthető: telepítse a teljes implementáció``pip`` vagy git segítségével
  • egyszerű YAML konfiguráció
  • könnyen bevethető: UbuntuGIS vagy a hivatalos Docker image segítségével
  • rugalmas: az egyedi adatkapcsolatok, formátumok és feldolgozások készítéséhez robusztus modul keretrendszerre épül
  • bármilyen Python web keretrendszert támogat (beleértve a Flask-et [alapértelmezett], Starlette-t)

Implementált szabványok¶

A szabványok a pygeoapi középpontjában vannak. Lentebb a projekt szabvány támogatási mátrixa van.

  • Implementálás: szabványt valósít meg (jó)
  • Megfelelőség: megfelel az OGC megfelelőségi követelményeknek (nagyszerű)
  • Referencia implementáció: referenciát biztosít a szabványhoz (fantasztikus!)
