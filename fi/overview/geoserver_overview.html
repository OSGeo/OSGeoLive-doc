GeoServer¶

Verkkopalvelu¶

GeoServer on verkkopalvelin, jonka avulla voit jakaa karttoja ja tietoja eri muodoissa eri sovelluksiin ja ohjelmiin, kuten web-selaimiin ja työasema GIS-ohjelmiin. Paikkatiedot julkaistaan OGC-standadien mukaisten rajapintaprotokollien, kuten WMS-, WFS-, WCS- ja WPS- avulla. GeoServeria voi käyttää sen selainpohjaisen käyttöliittymän avulla ja siihen voidaan yhdistää datoja laajasti eri tietoformaateista ja ohjelmistoista.

Päätoiminnallisuudet¶

Serve data from a variety of data stores:
  • Vektori

    • Shapefiles, ulkoinen WFS

    • PostGIS, ArcSDE, DB2, Oracle Spatial, MySql, SQL Server

  • Rasteri

    • GeoTiff, JPG ja PNG (+ world-tiedosto), karttapyramidi, GDAL-formaatit, karttamosaiikki, Oracle GeoRaster, NetCDF

** Data jaetaan tietoturvallisesti ja ketterästi WMS- ja WMTS-protokollia hyödyntäen: **

Koska paikkatiedot jaetaan kuvina, datan jakaminen on turvallista GeoServerin käytössä. Tietojesi varastaminen on näin erittäin haasteellista. Kartta-tasojen ulkoasua voidaan ohjata SLD-standardia hyödyntäen, mikä mahdollistaa värien ja nimiöiden käytön paikkatietojen visualisoinnissa. Myös OGC-filtterien avulla voidaan tehdä mittakaavakohtaista paikkatietojen visualisointia ja siten antaa käyttäjälle mahdollisuus nähdä zoomatessaan enemmän ja enemmän tietoja kartalla. Nimiöiden välisten yhteentörmäysten hallinta, ryhmitykset ja priorisointi on myös mahdollista toimittaa.

** Vektoritietoja voidaan lähettää kokonaisuudessaan asiakasohjelmistoihin WFS-protokollaa hyödyntäen: **

WFS-protokollaa lukevat paikkatieto-ohjelmistot ja -sovellukset voivat ladata ja käyttää vektoritietoja kartoituksen, spatiaalisten analyysien ja muiden paikkatieto-operaatioiden toteuttamiseen. WFS-T-protokollan avulla käyttäjät voivat myös muokata paikkatietoja palvelimella omien ohjelmistojensa avulla. Data voidaan siirtää palvelimelle hyödyntäen GML- (pakattu) ja muita yleisesti käytettyjä dataformaatteja, kuten shape- ja json-tiedostoformaatteja.

Raster data values can be sent to a client using the WCS protocol:

GIS-client-ohjelmisto voi hakea varsinaiset rasteriaineistojen arvot käytettäväkseen paikkatietojen analysoinnissa. Näin voidaan luoda sovelluksia, jotka voivat hyödyntää paikkatietojasi kuvatulla tavalla.

On the fly reprojection:

GeoServer tukee enemmistöä EPSG-tietokannan karttaprojektioista ja voi uudelleenprojektoida niihin lennossa, jotta rajallisen kapasiteetin omaavat käyttäjät voivat siirtää uudelleenprojektoinnista koituvan kuorman palvelimelle.

Tiling Cache

GeoWebCache, joka on integroitu GeoServeriin, laatat ja välimuistit kartoittävät kuvia useista eri lähteistä, kuten WMS:stä. Se toteuttaa erilaisia palvelurajapintoja (kuten WMS-C, WMTS, TMS, Google Maps KML, Virtual Earth), jotta voidaan nopeuttaa ja optimoida karttakuvan toimitus. Se voi myös yhdistää laatat toimimaan säännöllisesti WMS asiakkaita.

Web Processing Service (WPS)

Geospatiaalisia algoritmeja, mukaan lukien JTS, voidaan kutsua GeoServerin Web Processing Service (WPS) Käyttöliittymä. Tämä sisältää prosessien ketjutuksen ja renderointimuunnosten käyttämisen tehokkaiden visualisointien luomiseksi WMS-käyttöliittymän kautta.

Toteutetut standardit¶

Usean Open Geospatial Consortium (OGC) -pohjaisten standardien tukeminen:

  • Web Map Service (WMS). (asiakas/palvelin)

  • OpenGIS Web Map Tile ServiceW (WMTS)

  • Web Feature Service (WFS) (tapahtumallinen)

  • Web Coverage Service (WCS)

  • Filter Encoding (FE)

  • Styled Layer Descriptor (SLD)

  • Geography Markup Language (GML)

  • KML
