PROJ¶

PROJ es una biblioteca que proporciona métodos para transformar entre sistemas de referencia de coordenadas diferentes. Un sistema de referencia de coordenadas geográficas permite que todos los puntos de la tierra puedan ser descritos como un conjuntos de coordenadas (como latitud, longitud y altitud). Diversos sistemas se utilizan para representar la tierra tridimensional en un plano o mapa bidimensional.

PROJ se usa en MapServer, GRASS GIS, PostGIS, Mapnik, Thuban, OGDI, TopoCad, OGRCoordinateTransformation y varios otros.

PROJ ha sido traducido de C a otros idiomas. Estos subprotectos incluyen:

  • Proj4js (JavaScript) - usado en OpenLayers.

  • CS-map (C++) - utilizado en MapGuide.

  • GeoTIFF/libgeotiff (C++).

  • Proj4J (Java).

  • EPSG.io - website para obtener definiciones de sistemas de coordenadas.

  • SpatialReference.org - website para obtener definiciones de sistemas de coordenadas. (Obsoleto)

Características Principales¶

  • Proporciona el punto de transformación de un sistema de referencia de coordenadas a otro.

  • Incluye transformación entre los datos de referencia o datums.

  • Gran cantidad de clases de proyección compatibles.

Estándares Implementados¶

PROJ utiliza las definiciones del sistema de coordenadas dadas por el European Petroleum Survey Group (EPSG) <https://www.epsg.org/>. Bien conocido formato de texto (WKT).
