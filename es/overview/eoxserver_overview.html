EOxServer¶

Acerca de¶

EOxServer es un servidor para la presentación de grandes archivos de datos de observación de la tierra (EO) y metadatos a través de estándares abiertos.

Los datos deEarth Observation actualmente admitidos incluyen datos de trama 2D generalmente adquiridos usando sensores en satélites o aeronaves e incluyendo información sobre el momento de la adquisición y la huella en el planeta Tierra. Normalmente se usa para monitorear el entorno natural y construido de la Tierra.

EOxServer se basa en una pila de software de código abierto que incluye Python, MapServer, Django/GeoDjango, GDAL, PROJ, y un SpatiaLite o base de datos PostGIS .

Características Principales¶

  • Ver, filtrar, subconjunto y descargar datos de EO

  • Registro de archivos de datos EO

  • Interfaz de administración basado en la web y en línea de comandos

  • Publicar datos a través de Open Geospatial Consortium (OGC)

    • Web Coverage Service (WCS) 1.0, 1.1 y 2.0 & propuesto EO-WCS

    • Web Map Service (WMS) & EO-WMS

    • Propuesta de extensiones WCS incluyendo: codificación GeoTIFF, proyecciones predefinida (CRSs), e interpolación.

    • Los protocolos soportados son: KVP y XML/POST (utilizados junto con el proxy SOAP2POST para proporcionar también XML/SOAP)

  • Registrar datos almacenados en S3

  • Publicación utilizando los siguientes formatos y agrupaciones:

    • Coberturas 2-D EO derivadas de gmlcov:RectifiedGridCoverage

    • Coberturas 2-D EO derivadas de gmlcov:ReferenceableGridCoverage

    • Serie de conjunto de datos como una colección de coberturas de EO por ejemplo, en una serie de tiempo

    • Coberturas de mosaicos cosidos rectificados EO incluyendo contributingFootprint

  • Cobertura de formatos soportados:

  • GeoTIFF

  • Formatos soportados por la biblioteca GDAL

Estándares implementados¶

  • WCS, EO-WCS

  • WMS, EO-WMS

  • WPS

  • GML, GMLCOV, EO-O&M

  • OpenSearch
