pycsw¶

Catálogo de metadatos¶

pycsw permite a los usuarios publicar un catálogo de descripciones de conjunto de datos (metadatos) en la web, utilizando interfaces y formatos basados en estándares, de modo que pueda ser consultado y actualizado por los clientes del catálogo. pycsw es fácil de configurar dentro de un servidor web Apache (o cualquier servidor WSGI) y puede acceder a registros de metadatos almacenados como archivos XML dentro de una base de datos.

Los metadatos son “datos acerca de datos”, que almacenan información como fecha de creación, autor, título, área de interés, etc., y generalmente están codificados como archivos XML, siguiendo estándares internacionales como ISO 19115, ISO 19139 y FGDC CSDGM.

pycsw es una implementación Python de el Catalogue Service. Es un proyecto certificado por OGC Compliant, y es una OGC Reference Implementation. El estándar de CSW define interfaces comunes para descubrir, navegar y consultar metadatos sobre datos espaciales, sevicios web y recursos relacionados.

pycsw es de Código Abierto, publicado bajo una licencia MIT y se puede ejecutar en todas las plataformas principales (Windows, Linux, Mac OS X).

Características Principales¶

  • Certificación OGC Compliant y OGC Implementación de Referencia para OGC CSW 2.0.2 y CSW 3.0.0

  • apoyo a la recolección de metadatos para WMS, WFS, WCS, WPS, WAF, CSW, SOS

  • implementa INSPIRE Discovery Services 3.0

  • implementa ISO Metadata Application Profile 1.0.0

  • implementa FGDC CSDGM Application Profile para CSW 2.0

  • implementa la búsqueda/recuperación a través del protocolo de búsqueda URL (SRU)

  • implementa las Capacidades de Búsqueda de Texto

  • implementa extensiones de OGC OpenSearch Geo/Time

  • implementa el Protocolo de Iniciativa de Archivos Abiertos para la Recolección de Metadatos

  • Soporta modelos de metadatos ISO, Dublin Core, DIF, FGDC, Atom y GM03

  • despliegue de CGI o WSGI

  • Python 2 y 3 compatibles

  • configuración simple

  • capacidades transaccionales (CSW-T)

  • configuración de repositorio flexible

  • Conectividad GeoNode

  • Conectividad HHypermap

  • Conectividad de catálogo de Open Data

  • Conectividad CKAN

  • Búsqueda en catálogos federales distribuidos

  • validación en tiempo real de esquema XML

  • arquitectura de plugin de perfil extensible

  • muy ligero (Python + CGI)

Estándares implementados¶

Estándares OGC

  • Catalogue Service 2.0.2, 3.0.0

  • Filter Encoding (FE) 1.1.0, 2.0.0

  • OWS Common 1.0.0, 2.0.0

  • Geography Markup Language (GML) 3.1.1

  • Simple Feature Access (SFS) 1.2.1

  • OGC OpenSearch 1.0

Estándares de metadatos:

  • Dublin Core 1.1

  • SOAP 1.2

  • ISO 19115:2003

  • ISO 19139:2007

  • ISO 19119:2005

  • NASA DIF 9.7

  • FGDC CSDGM 1998

  • GM03 2.1

  • SRU 1.1

  • OAI-PMH 2.0

  • A9 OpenSearch 1.1
