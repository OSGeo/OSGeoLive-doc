MapCache¶

Servicio Web¶

MapCache es un servidor que implementa el almacenamiento en caché de mosaicos para acelerar el acceso a las capas WMS. Los objetivos principales son ser rápidos y fáciles de implementar, al tiempo que ofrecen las características esenciales (¡y más!) que se esperan de una solución de almacenamiento en caché de teselas.

MapCache es parte de la familia MapServer, pero se puede configurar para almacenar en caché y servir datos de cualquier servidor geoespacial que use WMS. MapCache se puede ejecutar en Apache como un módulo o en cualquier servidor web que admita CGI/FastCGI como IIS, Nginx y Jetty.

Características Principales¶

  • Allows tile caches to be stored in
    • en disco
    • en la nube usando REST (S3, Azure, Google)
    • en bases de datos que incluyen SQLite, Berkeley DB, Riak
    • en memoria cache
  • Incluye una aplicación semillera
  • Cache basado en dimensiones (como tiempo)
  • Permite el almacenamiento en caché de mosaicos Vector Mapbox (MVT) y UTFGrid
  • Cuadrículas personalizables
  • Permite que los mosaicos se «ensamblen» juntos, lo que permite combinar mosaicos para diferentes conjuntos de datos en un solo mosaico para enviar al cliente
  • Incluye una aplicación de semillera para llenar previamente cachés desde la línea de comando
  • Permite que las solicitudes se devuelvan a un servidor
  • na interfaz de demostración incorporada

Estándares implementados¶

  • Apoyo de numerosos estándares del Open Geospatial Consortium (OGC)
    • Web Map Service (WMS)
    • OpenGIS Web Map Tile ServiceW (WMTS)
  • Estándares adicionales incluyen:
    • TMS
    • Google Maps XYZ
    • SuperOverlays KML
