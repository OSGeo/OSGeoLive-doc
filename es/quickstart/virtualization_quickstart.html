Inicio Rápido de OSGeoLive para Ejecutar en una Máquina Virtual¶

Este Inicio Rápido describe una forma de ejecutar OSGeoLive dentro de una máquina virtual de VirtualBox. Para otros métodos, siga los enlaces de la sección «Ver también» que se muestra a continuación.

Ventajas de la Máquina Virtual¶

  • La velocidad de respuesta de una VM es mucho más rápida que en el DVD, y generalmente es comparable con la instalación directamente en la máquina host.
  • Todos los cambios de configuración, actualizaciones de software y archivos guardados son permanentes, al igual que en cualquier equipo «normal».
  • Puede personalizar y actualizar el sistema
  • Usted puede guardar su trabajo dentro de la Máquina Virtual
  • Puede instalar software adicional
  • Usted puede hacer una copia de seguridad de la instalación de su máquina virtual

Requisitos del Sistema¶

  • RAM: 2 GB por lo menos. El sistema Lubuntu system funciona bien con 1 GB de RAM, y usted sólo necesita matener al menos la misma cantidad de memoria en su sistema host. Por lo tanto, se necesitarán un total de 2 GB para un funcionamiento sin problemas. Algunas aplicaciones, como geoserver, necesitan que la máquina virtual tenga al menos 2 GB de RAM. De ser posible, considere proporcionar 2 GB o más para su máquina virtual.
  • Spare Hard Disk Space: The virtual disk file (vmdk) from: live.osgeo.org unzips to almost 10 GB. And you’ll want more space to allow some room to work on the virtual machine. So 20 GB is a good recommendation.
  • CPU: Any CPU will do, but a processor which supports «Virtualization Technology» will be faster. You can check your computer CPU here.

La mayoría de los equipos producidos en los últimos 4-5 años cumplirá estos requisitos.

Cómo¶

Descargas

Download Virtual Box for your platform, and install the software. On windows run the installer, or on Ubuntu-like linux systems do the following:

apt-get install virtualbox-ose

In addition download the OSGeo Virtual disk file (vmdk) from live.osgeo.org . Click on the «Download virtual machine» button. Then unzip the downloaded file (using 7zip ).

** Crear una Máquina Virtual **

Iniciar la aplicación VirtualBox y haga clic en el botón nuevo para crear una nueva MV, y luego Siguiente.

Introduzca un nombre, por ejemplo, OSGeoLive, elija Linux como «Sistema Operativo» y Ubuntu como la «Versión».

En la siguiente pantalla, configure la memoria en 1024 MB (o más si su computadora host tiene más de 4 GB, como en la captura de pantalla).

Continúe a la pantalla siguiente y elija «Usar disco duro existente». Ahora haga clic en el botón (icono de una carpeta) para buscar el lugar donde guardó el archivo vmdk de OSGeoLive. Seleccione este archivo, presione Siguiente y Crear.

** Configuración consejos y trucos **

Una vez que se crea la VM, haga clic en el botón Configuración. En la sección «General», vaya a la pestaña Avanzado y haga clic para seleccionar «Mostrar en la parte superior de la pantalla» para la Mini barra de herramientas.

Vaya a la sección «Display» y aumente la memoria de vídeo a 32 o 64 MB. En algunos sistemas operativos huéspedes, puede ser necesario «Habilitar aceleración 3D» para que la pantalla funcione correctamente.

Además, vaya a la sección «Carpetas compartidas» y haga clic en «Agregar carpeta» (icono verde + a la derecha) para buscar un directorio en su computadora host que desee compartir dentro de la VM.

Puede seleccionar que la carpeta compartida sea de sólo lectura y se monte automáticamente. Una vez que se definan la «Ruta de Carpeta» y el «Nombre de Carpeta», haga clik en OK, y de nuevo OK para finalizar y cerrar la ventana de configuraciones.

** Ejecutando la Máquina Virtual **

Ahora, inicie la VM haciendo clik en el botón de Start o Inicio (flecha verde).

Al instalar el iso en la máquina virtual, el nombre de usuario debe establecerse en «user»

El usuario «user» no es miembro del grupo «users» en la máquina virtual OSGeoLive. Esto evita que algunas herramientas escriban en su directorio de datos y cause algún software falle al iniciar.

La solución consiste en corregir el vmdk con el siguiente procedimiento que se hace una sola vez:

  1. Inicie una terminal.
  2. Ejecute «sudo adduser user users».
  3. Aplique este cambio iniciando una nueva sesión de escritorio: reinicie la máquina virtual o cierre la sesión y vuelva a iniciarla (nombre de usuario «user», contraseña «user»).

Una vez que el sistema de OSGeo arranca, agréguese al grupo de vboxsf para que las carpetas compartidas (definidas anteriormente) sean accesibles ejecutando una ventana de terminal

user@osgeolive:~$ sudo usermod -a -G vboxsf user

En el ejemplo anterior, definimos la ruta de una Carpeta Compartida n the above example, we defined a Shared Folder path en el sistema host y lo llamamos «GIS» en Configuraciones de VM. La carpeta compartida aparecerá en el sistema de archivos en /media/sf_GIS/. Para montar esta carpeta en el directorio de inicio del usuario, por ejemplo, en un terminal hacer lo siguiente:

user@osgeolive:~$ mkdir GIS

user@osgeolive:~$ sudo mount -t vboxsf -o uid=user,rw GIS /home/user/GIS

Ahora, los archivos en su máquina host, en la carpeta elejida, estarán disponibles también en la VM, dentro del folder «GIS» en el directorio de inicio del usuario.
