Inicio Rápido de OSGeoLive para Ejecutar en una Máquina Virtual¶

Este Inicio Rápido describe una forma de ejecutar OSGeo-Live en una Máquina Virtual de VirtualBox. Para otros métodos, siga los enlaces de la sección «Vea también» más abajo.

Ventajas de la Máquina Virtual¶

  • The response speed of a VM is much faster than on the DVD, and typically comparable with installing directly on the host machine.
  • All configuration changes, software updates, and saved files are permanent, just like on any «regular» computer. So:
  • Puede personalizar y actualizar el sistema
  • Usted puede guardar su trabajo dentro de la Máquina Virtual
  • Puede instalar software adicional
  • Usted puede hacer una copia de seguridad de la instalación de su máquina virtual

Requisitos del Sistema¶

  • RAM: 2 GB at least. The Lubuntu system runs well with 1 GB of RAM, and you’ll need to keep at least the same amount of memory for your host system. So a total of 2 GB will be needed for smooth operation. Some applications, like GeoServer, needs the virtual machine to have at least 2GB of RAM. So if possible, consider providing 2 GB or more for your virtual machine.
  • Spare Hard Disk Space: The virtual disk file (vmdk) from live.osgeo.org unzips to almost 10 GB. You’ll also want more space to allow some room to work on the virtual machine. So 20 GB is a good recommendation.
  • CPU: Cualquier CPU funcionará, pero un procesador que soporte «Virtualization Technology» será más rápido. Puede revisar el CPU de su computadora aquí.

La mayoría de los equipos producidos en los últimos 4-5 años cumplirá estos requisitos.

Cómo¶

Descargas

Download Virtual Box for your platform, and install the software. On Windows run the installer, or on Ubuntu-like Linux systems do the following:

apt-get install virtualbox-ose

Adicionalmente, descargue el Archivo de Disco Virtual (vmdk) de OSGeo desde live.osgeo.org . Haga clik en el botón «Descargar máquina virtual». Después, descomprima el archivo descargado (usando 7zip ).

** Crear una Máquina Virtual **

Iniciar la aplicación VirtualBox y haga clic en el botón nuevo para crear una nueva MV, y luego Siguiente.

Introduzca un nombre, por ejemplo, OSGeoLive, elija Linux como «Sistema Operativo» y Ubuntu como la «Versión».

In the next screen set the memory to 1024 MB (or more if your host computer has more than 4GB, like in the screenshot below).

Continue to the next screen and choose «Use an existing virtual hard disk file». Now click on the button (a folder icon) to browse to where you saved the OSGeoLive vmdk-file. Select this file, press Next and Create.

Config tips and tweaks

Una vez que se crea la VM, haga clic en el botón Configuración. En la sección «General», vaya a la pestaña Avanzado y haga clic para seleccionar «Mostrar en la parte superior de la pantalla» para la Mini barra de herramientas.

Vaya a la sección «Display» y aumente la memoria de vídeo a 32 o 64 MB. En algunos sistemas operativos huéspedes, puede ser necesario «Habilitar aceleración 3D» para que la pantalla funcione correctamente.

Además, vaya a la sección «Carpetas compartidas» y haga clic en «Agregar carpeta» (icono verde + a la derecha) para buscar un directorio en su computadora host que desee compartir dentro de la VM.

You can choose to make the shared folder read-only, and auto-mounted. Once the «Folder path» and «Folder name» are defined, click OK, and again OK to finish and close the settings window.

Ejecutando la Máquina Virtual

Ahora, inicie la VM haciendo clikk en el botón Iniciar (flecha verde).

Al instalar el iso en la máquina virtual, el nombre de usuario debe establecerse en «user»

El usuario «user» no es miembro del grupo «users» en la máquina virtual OSGeoLive. Esto evita que algunas herramientas escriban en su directorio de datos y cause algún software falle al iniciar.

La solución consiste en corregir el vmdk con el siguiente procedimiento que se hace una sola vez:

  1. Inicie una terminal.
  2. Ejecute «sudo adduser user users».
  3. Aplique este cambio iniciando una nueva sesión de escritorio: reinicie la máquina virtual o cierre la sesión y vuelva a iniciarla (nombre de usuario «user», contraseña «user»).

Also once the OSGeo system starts, add yourself to the vboxsf group so that the shared folders (defined above) are accessible by running in a terminal window:

user@osgeolive:~$ sudo usermod -a -G vboxsf user

In the above example, we defined a Shared Folder path on the host system and named it «GIS» in the VM Settings. The shared folder will appear in the file system under /media/sf_GIS/. To mount this folder in the user’s home directory, for example, in a terminal run:

user@osgeolive:~$ mkdir GIS

user@osgeolive:~$ sudo mount -t vboxsf -o uid=user,rw GIS /home/user/GIS

Ahora, los archivos en su máquina host, en la carpeta elejida, estarán disponibles también en la VM, dentro del folder «GIS» en el directorio de inicio del usuario.
