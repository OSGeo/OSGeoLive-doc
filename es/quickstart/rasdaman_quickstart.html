Inicio Rápido de Rasdaman¶

Rasdaman es un Gran Motor de Datos para análisis ad-hoc y flexibles de datos espacio-temporales multidimensionales de sensores, imágenes, simulaciones y estadísticas de tamaño ilimitado. El lenguaje de consulta de Servicio Web de Procesamiento de Cobertura (WCPS) es un estándar de Open Geospatial Consortium (OGC) que permite filtrado y procesamiento de coberturas ráster múltiples dimensiones, tales como sensores, simulación, imagen y estadísticas de datos mediante servicios web. Las consultas WCPS se traducen a lenguaje de interrogación rasdaman, rasql y se ejecutan en rasdaman. Esta guía de inicio rápido muestra cómo tener acceso y manipular un ejemplo de cobertura 2D usando WCPS lenguaje.

Configuración¶

El primer paso antes de probar cualquiera de las siguientes consultas es iniciar rasdaman y Tomcat. Abra el directorio Databases en el escritorio y, a continuación, Start Rasdaman Server. Espere de dos a tres minutos para que rasdaman y especialmente Tomcat se inicien por completo (una vez que el cliente web rasdaman se carga en el navegador).

Después, puede abrir Rasdaman-Earthlook Demo desde el mismo directorio, que lanzará un demostrador local en el navegador. Para obtener un tutorial más práctico, sin embargo, continúe con los siguientes ejemplos de aquí.

Running WCPS queries¶

Ya sea:
  • Vaya al cliente web rasdaman, cambie a la pestaña ProcessCoverages e introduzca su consulta en los cuadros de consulta y presione enviar.
  • Agregue su consulta en esta url: http://localhost:8080/rasdaman/ows?query=your_query y ejecútela usando un navegador.

Access full coverage¶

La consulta WCPS para acceder a la cobertura completa es como sigue:

for c in (NIR) return encode(c, "png")

haga click en coverage request para ejecutarlo en el navegador.

Select a subset of the coverage¶

La consulta WCPS a un subconjunto de la cobertura es la siguiente:

for c in (NIR) return encode(c[i(0:500),j(0:500)], "png")

Haga click en la consulta subsequente para ejecutarlo en el navegador.

Band extraction from the coverage¶

La consulta WCPS para accesar a la banda roja de la cobertura es la siguiente:

for c in (NIR) return encode(c.red, "png")

Arithmetic operation: Extracting NDVI of the coverage¶

El NDVI (Índice de Vegetación de Diferencia Normalizada) es una medida de la probabilidad de la vegetación en teledetección, es decir, cuanto más cerca esté de +1 un píxel, más probable es que sean plantas. La consulta de WCPS para obtener NDVI de la cobertura es la siguiente:

for c in ( NIR ) return
encode(
  (unsigned char) (
     (((float)c.0 - (float)c.1) /
      ((float)c.0 + (float)c.1)) > 0
  ) * 255
, "png" )

¿Qué sigue?¶

  • El tutorial “tutorial de rasdaman <http://tutorial.rasdaman.org>`_ proporciona una introducción condensada a los conceptos detrás de rasdaman, rasql y al soporte de servicios web OGC.
  • Aprenda más acerca del Lenguaje Estándar (WCPS) de OGC Web Coverage Processing Service (que está fuertemente inspirado en rasdaman); un buen punto de partida es la página de documentación y tutoriales y WCPS Query SandBox.
  • ¿Interesado en cobetura de datos y servicios en general? Visite OGC’s Coverages Domain Working Group wiki.
  • Explore rasdaman documentation para aprender sobre la administración de rasdaman, su lenguaje de consultas e ingestión de datos.
  • Vaya a multi-dimensional demo para explorar los demos interactivos 1-D a 4-D de rasdaman. Los más pequeños son parte de OSGeoLive, aquellos que utilizan conjuntos más grandes (más allá de la capacidad de este disco), avanzan hacia el OGC standards showcase site .
